# Proposal Source Book Quality Rubric

> Derived from analysis of 3 real Strategic Gears proposals (examples 1-3).
> Domain-agnostic and reusable across sectors.

## Scoring Scale

Each dimension is scored 1-5:

| Score | Label | Meaning |
|-------|-------|---------|
| 5 | Benchmark | Matches or exceeds the best real example |
| 4 | Strong | Minor gaps vs. real examples |
| 3 | Adequate | Functional but noticeably weaker than examples |
| 2 | Weak | Significant gaps; not evaluator-ready |
| 1 | Unacceptable | Missing, placeholder, or materially wrong |

## Dimensions

### D1: RFP Understanding Depth (Weight: 10%)

**What the real examples do:**
- Map project objectives to national strategy (Vision 2030 pillars, DGA standards)
- Show awareness of regulatory bodies and compliance frameworks
- Identify unstated evaluator priorities (Saudization, local content)
- Include a compliance alignment matrix (RFP requirement -> response location)
- Demonstrate prior knowledge of the client's context

**Scoring criteria:**
- 5: Explicit mapping to national strategy + regulatory awareness + compliance matrix + client context
- 4: Most of the above, minor gaps
- 3: General understanding, missing compliance matrix or national strategy mapping
- 2: Generic RFP summary, no client-specific insight
- 1: Missing or placeholder

### D2: Methodology Depth and Specificity (Weight: 20%)

**What the real examples do:**
- 40-50 slides of methodology in the deck (the largest section)
- 4-5 distinct phases with named sub-stages
- Each phase has: activities (3-8 bullet items), deliverables, governance touchpoints
- References recognized frameworks (TOGAF, ITIL, PMBOK, COBIT, Agile, Lean Six Sigma)
- References national methodologies where applicable (NORA, DGA Qiyas)
- Includes illustrative examples from prior projects
- Maps phases to RFP requirements with traceability
- Includes demand management, change management, knowledge transfer components
- Benchmarking methodology with selection criteria for comparisons

**Scoring criteria:**
- 5: 5+ phases, each with 3+ sub-stages, named deliverables, framework references, RFP traceability
- 4: 4+ phases with deliverables and some framework references
- 3: 3+ phases with basic activities/deliverables
- 2: Generic methodology description without specificity
- 1: Missing or single-paragraph description

### D3: Governance Specificity (Weight: 10%)

**What the real examples do:**
- Dedicated governance section (10-11 slides in deck)
- Steering committee structure with meeting cadences
- RACI matrices per workstream
- Escalation procedures with named levels
- Monthly operational and quarterly strategic review cadences
- Project reporting templates and KPI dashboards
- Change request management process
- Risk management framework with risk register

**Scoring criteria:**
- 5: Full governance framework with RACI, escalation, cadences, reporting, risk management
- 4: Most governance elements present
- 3: Basic governance structure (steering committee + reporting)
- 2: Single paragraph about governance
- 1: Missing

### D4: Consultant Profiling Quality (Weight: 15%)

**What the real examples do:**
- 12-15 named team members per proposal
- Each profile includes: full name, current role, years of experience, specific certifications (PMP, TOGAF, CISSP, SAFe, etc.), prior employers (Deloitte, PwC, McKinsey), education (university, degree)
- Clear role assignment: Project Director, Engagement Manager, Technical Lead, SMEs
- Team org chart showing reporting structure
- Saudization ratio disclosed
- Each person's relevance to the specific project stated

**Scoring criteria:**
- 5: 10+ named people with credentials, certifications, years, prior employers, role assignments, org chart
- 4: 8+ named people with most credential details
- 3: 5+ named people with basic role/experience info
- 2: Generic team description with few names or placeholder profiles
- 1: Missing or entirely placeholder

### D5: Prior Project Relevance and Evidence Density (Weight: 15%)

**What the real examples do:**
- 10-12 case studies with named clients
- Each case study includes: client name, project title, sector, challenge, SG contribution, measurable outcomes
- Organized by sector or relevance theme
- Include client logos as visual proof
- Quantified outcomes where possible (e.g., "portfolio >$100M", "240+ projects")
- Prior work in the same sector/domain as the RFP highlighted

**Scoring criteria:**
- 5: 8+ named case studies with client, challenge, contribution, outcomes, sector match
- 4: 6+ case studies with most detail
- 3: 3-5 case studies with basic info
- 2: Vague references to prior work without named clients
- 1: Missing or entirely fabricated

### D6: Compliance-to-RFP Mapping Quality (Weight: 5%)

**What the real examples do:**
- Explicit compliance table/checklist
- Each RFP requirement mapped to proposal response section
- Compliance status (compliant/partially compliant/alternative proposed)
- Regulatory/legal compliance declarations
- Conflict of interest declarations

**Scoring criteria:**
- 5: Full compliance matrix with requirement-to-section mapping + regulatory declarations
- 4: Compliance table with most requirements mapped
- 3: General compliance statement without detailed mapping
- 2: Single compliance paragraph
- 1: Missing

### D7: Slide-by-Slide Recommendation Quality (Weight: 10%)

**What the real examples do (Source Book Section 6):**
- One entry per proposal slide with: purpose, title, key message, bullet logic, evidence references, visual guidance
- Blueprints cover ALL standard sections (Cover, TOC, Understanding, Why SG, Methodology, Timeline, Team, Case Studies, Governance, Company Profile)
- Each blueprint entry specifies which evidence must appear on the slide
- Blueprint entries specify forbidden content (generic claims to avoid)

**Scoring criteria:**
- 5: 20+ blueprint entries covering all sections with evidence mapping
- 4: 15+ entries covering most sections
- 3: 8-14 entries with basic guidance
- 2: Fewer than 8 entries, minimal guidance
- 1: Empty or missing

### D8: Evidence Ledger Completeness (Weight: 10%)

**What the Source Book must contain (Section 7):**
- Every CLM-xxxx and EXT-xxx ID cited anywhere in sections 1-6 must appear
- Each entry: claim_id, claim_text, source_type, source_reference, confidence (0-1), verifiability_status
- Cross-references verified: no orphan citations, no undeclared IDs
- Mix of internal (CLM) and external (EXT) evidence types

**Scoring criteria:**
- 5: 15+ entries with all fields populated, all citations cross-referenced, mix of internal/external
- 4: 10+ entries with most fields, minor orphan citations
- 3: 5-9 entries, some missing cross-references
- 2: 1-4 entries, mostly incomplete
- 1: Empty

### D9: Executive Tone / Evaluator Readiness (Weight: 5%)

**What the real examples do:**
- Professional Arabic (or English) language appropriate for government evaluators
- Formal cover letter signed by named partners
- Clear section numbering and navigation (TOC)
- Consistent formatting and structure
- No placeholder text, no [brackets], no TODO markers
- Evidence-backed claims (not fluff)
- Quantified metrics where possible

**Scoring criteria:**
- 5: Professional tone, zero placeholders, all claims backed, formal structure
- 4: Professional with minor issues
- 3: Mostly professional but some placeholder text or unsupported claims
- 2: Significant fluff, placeholder text, or unprofessional language
- 1: Draft-quality with major issues

## Aggregate Scoring

**Weighted total = sum of (dimension_score * weight)**

| Rating | Weighted Score | Meaning |
|--------|---------------|---------|
| Benchmark | >= 4.5 | Ready to submit |
| Strong | 4.0 - 4.4 | Minor polish needed |
| Adequate | 3.0 - 3.9 | Usable but needs work |
| Weak | 2.0 - 2.9 | Not evaluator-ready |
| Unacceptable | < 2.0 | Must be rewritten |

## Hard Gates (Automatic Rejection)

The Source Book is automatically rejected if ANY of these are true, regardless of aggregate score:

1. Section 6 (Slide Blueprint) is empty or has fewer than 8 entries
2. Section 7 (Evidence Ledger) is empty
3. External evidence artifacts are missing (no research was attempted)
4. Named consultants remain placeholders where KG data exists
5. Prior projects are placeholder-level where KG data exists
6. Compliance mapping is missing when the RFP contains compliance requirements
7. Any section contains [bracket] placeholders or TODO markers
